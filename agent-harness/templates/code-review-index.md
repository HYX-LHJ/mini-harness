# code_review — 审查留底

本回合改过业务代码时由 Subagent 审查；主 Agent **必须**将报告落盘本目录。

**命名**：`YYYY-MM-DD_主题简述.md`（日期在前）。

未修 **P0/P1** 登记 [open-findings.md](open-findings.md)。

## 严重度约定

| 级别 | 含义 |
|------|------|
| P0 | 安全/数据丢失，阻塞合并 |
| P1 | 逻辑错误或显著架构问题 |
| P2 | 可维护性 / 扩展性 |
| P3 | 风格与小幅优化 |
