# 删除与迭代计划模板

## 优先级

- [ ] **P0**：需立即删除（安全风险、显著成本、阻塞其他工作）
- [ ] **P1**：在当前迭代中删除
- [ ] **P2**：待办 / 下一迭代

---

## 可立即安全删除

### 项目：[名称/描述]

| 字段 | 详情 |
|-------|---------|
| **位置** | `path/to/file.ts:line` |
| **理由** | 为何应删除 |
| **证据** | 未使用（无引用）、已死功能开关、已弃用 API |
| **影响** | 无 / 低 — 无活跃消费者 |
| **删除步骤** | 1. 删除代码 2. 删除测试 3. 删除配置 |
| **验证** | 运行测试、确认无运行时错误、监控日志 |

---

## 延后删除（需制定计划）

### 项目：[名称/描述]

| 字段 | 详情 |
|-------|---------|
| **位置** | `path/to/file.ts:line` |
| **延后原因** | 仍有活跃消费者、需迁移、需利益相关方签字 |
| **前置条件** | 功能开关关闭 2 周、遥测显示使用量为 0 |
| **破坏性变更** | 列出任何 API/契约变更 |
| **迁移计划** | 消费者迁移步骤 |
| **时间线** | 目标日期或迭代 |
| **负责人** | 负责的个人/团队 |
| **验证** | 确认可安全删除的指标（错误率、使用次数） |
| **回滚计划** | 发现问题后如何恢复 |

---

## 删除前检查清单

- [ ] 已搜索代码库中所有引用（`rg`、`grep`）
- [ ] 已检查动态/反射式用法
- [ ] 已确认无外部消费者（API、SDK、文档）
- [ ] 已审查功能开关遥测（如适用）
- [ ] 已更新/删除测试
- [ ] 已更新文档
- [ ] 已通知团队（若为共享代码）
