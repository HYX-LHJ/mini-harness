# 代码质量检查清单

## 错误处理

### 需标记的反模式

- **吞掉异常**：空 catch 块或 catch 中仅记录日志
  ```javascript
  try { ... } catch (e) { }  // 静默失败
  try { ... } catch (e) { console.log(e) }  // 记录后遗忘
  ```
- **过于宽泛的 catch**：捕获 `Exception`/`Error` 基类而非具体类型
- **错误信息泄露**：向用户暴露堆栈跟踪或内部细节
- **缺失错误处理**：可失败操作（I/O、网络、解析）周围无 try-catch
- **异步错误处理**：未处理的 Promise 拒绝、缺失 `.catch()`、无错误边界

### 需检查的最佳实践

- [ ] 在适当边界捕获错误
- [ ] 错误消息对用户友好（不暴露内部细节）
- [ ] 错误日志包含足够调试上下文
- [ ] 异步错误得到正确传播或处理
- [ ] 可恢复错误定义了回退行为
- [ ] 严重错误触发告警/监控

### 自问
- 「该操作失败时会发生什么？」
- 「调用方能否知道出了问题？」
- 「是否有足够上下文调试此错误？」

## 性能与缓存

### CPU 密集操作

- **热路径中的昂贵操作**：循环中的正则编译、JSON 解析、加密
- **阻塞主线程**：同步 I/O、无 worker/异步的重计算
- **不必要的重复计算**：相同计算多次执行
- **缺失记忆化**：纯函数以相同输入被反复调用

### 数据库与 I/O

- **N+1 查询**：循环中对每项单独查询而非批量
  ```javascript
  // 差：N+1
  for (const id of ids) {
    const user = await db.query(`SELECT * FROM users WHERE id = ?`, id)
  }
  // 好：批量
  const users = await db.query(`SELECT * FROM users WHERE id IN (?)`, ids)
  ```
- **缺失索引**：在未索引列上查询
- **过度获取**：仅需少量列却 SELECT *
- **无分页**：将整个数据集加载到内存

### 缓存问题

- **昂贵操作缺失缓存**：重复的 API 调用、数据库查询、计算
- **缓存无 TTL**：无限期提供陈旧数据
- **缓存无失效策略**：数据已更新但缓存未清除
- **缓存键冲突**：键唯一性不足
- **全局缓存用户专属数据**：安全/隐私问题

### 内存

- **无界集合**：无限增长的数组/映射
- **大对象长期持有**：持有引用阻止 GC
- **循环中字符串拼接**：应使用 StringBuilder/join
- **完整加载大文件**：应使用流式处理

### 自问
- 「该操作的时间复杂度是多少？」
- 「数据量 10 倍/100 倍时行为如何？」
- 「该结果是否可缓存？是否应该缓存？」
- 「能否批量处理而非逐项处理？」

## 边界条件

### Null/Undefined 处理

- **缺失 null 检查**：访问可能为 null 的对象属性
- **真值/假值混淆**：`0` 或 `""` 为有效值时使用 `if (value)`
- **过度使用可选链**：`a?.b?.c?.d` 掩盖结构问题
- **null 与 undefined 混用**：无明确约定地混用

### 空集合

- **未处理空数组**：代码假定数组有元素
- **空对象边界情况**：对空对象使用 `for...in` 或 `Object.keys`
- **访问首尾元素**：未检查长度就使用 `arr[0]` 或 `arr[arr.length-1]`

### 数值边界

- **除零**：除法前未检查
- **整数溢出**：大数超出安全整数范围
- **浮点比较**：使用 `===` 而非 epsilon 比较
- **负值**：索引或计数不应为负
- **差一错误**：循环边界、数组切片、分页

### 字符串边界

- **空字符串**：未作为边界情况处理
- **仅空白字符串**：通过真值检查但实际为空
- **超长字符串**：无长度限制导致内存/显示问题
- **Unicode 边界情况**：表情符号、RTL 文本、组合字符

### 需标记的常见模式

```javascript
// 危险：无 null 检查
const name = user.profile.name

// 危险：未检查就访问数组
const first = items[0]

// 危险：未检查就除法
const avg = total / count

// 危险：真值检查排除有效值
if (value) { ... }  // 对 0、""、false 失败
```

- 「若为 null/undefined 会怎样？」
- 「若集合为空会怎样？」
- 「该数字的有效范围是什么？」
- 「在边界处（0、-1、MAX_INT）会发生什么？」
